# FinLit Lab Security & Hardening Summary

## Firebase Authentication 
- **Anonymous Base**: Users are signed in silently in the background upon application initialization via `AuthService`. This immediately maps a unique `UID` to their session, enabling Firestore reads and writes without adding friction to the beginner-friendly learning experience.
- **Upgradeable Accounts**: The service exposes a `linkWithGoogle` adapter, allowing users to attach permanent Google credentials to their anonymous UID natively, carrying over all their quiz progress unharmed.

## Firestore Security Rules
All database queries are scoped strictly to the current `UID`. Broad read/writes are outright denied by default.
### Implemented Rules:
1. **`mastery/{userId}`**
   - **Allowed**: `allow read, write: if isSignedIn() && request.auth.uid == userId;`
   - **Rationale**: Students can only view or modify their personal educational scores.

2. **`quiz_attempts/{attemptId}`**
   - **Allowed**: Read/Create restricted strictly to the user matching `request.resource.data.userId == request.auth.uid`. Updates and Deletes are explicitly `false`.
   - **Rationale**: An attempt ledger requires immutability. No user (or hacker) should be able to alter their past scores to falsify mastery later.

3. **`simulations/{userId}`**
   - **Allowed**: Read/Write locked to `request.auth.uid == userId`.
   - **Rationale**: Any personalized numeric arrays generated by simulators are strictly isolated per user.

## App Check Enforcement
`@angular/fire/app-check` has been included in the compilation bundle. Note that for deployment, valid Google ReCaptcha Enterprise / reCAPTCHA v3 keys must be placed inside the `app.config.ts` provider array. Emulators actively bypass this check in the `environment.development.ts` state.

## AI Tutor Protections
- Financial boundaries are securely enforced by the `AI_SYSTEM_PROMPT` housed centrally in code memory.
- Current constraints explicitly block personalized advice matching arrays (`"recommend a stock", "what should I buy"`). This provides high abuse-resistance against generating illicit finance advice.
